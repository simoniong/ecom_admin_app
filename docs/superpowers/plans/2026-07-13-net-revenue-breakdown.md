# Net Revenue Breakdown Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Net Revenue breakdown to the dashboard — Gross Revenue, Refunds, Tax (net), Transaction Fees, and Net Revenue — so the company sees the money it actually keeps, not just Shopify's gross order total.

**Architecture:** Persist four new per-day breakdown columns on `shopify_daily_metrics`. `ShopifyAnalyticsService` already computes gross and refunds inside a single `revenue` number; split out the tax and fee components (fees come inline from Shopify's `OrderTransaction.fees`, which is Shopify-Payments-only). `DashboardMetricsService` aggregates the new columns and derives `net_revenue`. The existing `revenue` column and every metric derived from it (gross_profit, roas, margins) stay byte-for-byte unchanged. Five new metric cards render on the dashboard.

**Tech Stack:** Rails 8.1, PostgreSQL (UUID PKs), RSpec + FactoryBot, WebMock (stubbed Shopify GraphQL), Shopify Admin GraphQL API 2024-10, Hotwire/Turbo, Tailwind, i18n (en / zh-TW / zh-CN).

## Global Constraints

- Never commit to `main` or `staging`; work on `feature/net-revenue-breakdown` (already checked out).
- All table IDs are UUIDs (`id: :uuid, default: -> { "gen_random_uuid()" }`).
- Tests: RSpec + FactoryBot, no fixtures, no mocks (hit real DB; stub only external HTTP via WebMock). 95%+ coverage. Model + request + system specs required.
- Money columns: `decimal, precision: 12, scale: 2, default: 0, null: false`.
- **Do NOT change the meaning of `shopify_daily_metrics.revenue` or any metric derived from it** (`gross_profit`, `avg_order_value`, `roas`, `gross_margin_pct`, `net_profit`, `net_margin_pct`).
- Net Revenue formula: `net_revenue = gross_revenue - refunds - total_tax - transaction_fees`, where `total_tax` is **net tax** (`tax_charged - tax_refunded`).
- i18n: add keys to all three locale files under `dashboard:` — `config/locales/en.yml`, `config/locales/zh-TW.yml`, `config/locales/zh-CN.yml`. Never hardcode UI strings.

    min_time = @timezone.parse(date.to_s).beginning_of_day.utc

    refunds_total, tax_refunded = fetch_refunds_via_graphql(client, min_time, max_time)

    metric = ShopifyDailyMetric.find_or_initialize_by(
      shopify_store_id: @store_id, date: date
    )
    metric.assign_attributes(

      orders_count: orders_count,

      revenue: gross_revenue - refunds_total,
      gross_revenue: gross_revenue,
      refunds: refunds_total,
      total_tax: tax_charged - tax_refunded,
      transaction_fees: transaction_fees,
      conversion_rate: 0
